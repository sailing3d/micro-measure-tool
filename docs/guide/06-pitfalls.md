# 第六章: 常见坑与怎么办

遇到问题不要慌。大部分问题都是模式化的, 有固定的处理方式。

## 1. 拖拽或缩放后位置乱跳

**你看到的**: 拖图片到另一个格子, 或者缩放后, 图片位置和预期不一样——明明是拖到中间, 松手后飞到角落去了。

**怎么办**: 告诉 AI: "位置跳动, 怀疑是坐标系不一致, 检查相对坐标和绝对坐标的换算"。AI 自己知道怎么查。本项目这个问题修了 10+ 次, 最终发现是 `node.x()` 和 `getAbsolutePosition()` 混用导致的。

**关键信号**: 只要出现"位置跳"、"不居中"、"拖放后乱跑", 就是坐标问题。

## 2. 页面白屏或卡死

**你看到的**: 打开页面后浏览器卡住, 控制台报 "Maximum update depth exceeded", 页面完全不能用。

**怎么办**: 告诉 AI: "页面白屏, 有死循环"。这通常是 AI 最近的某次改动引入的——比如一个函数本该返回值, 却返回了新对象, 导致无限重渲染。

**关键信号**: 白屏 + console 里有 "Maximum update" 字样。

## 3. 刷新后数据丢了

**你看到的**: 刷新浏览器, 图片没了、测量结果没了、格子配置回到了默认值。

**怎么办**: 告诉 AI: "刷新后状态丢失, 需要持久化"。AI 会加数据保存功能 (本项目用了 IndexedDB 存项目指针, sessionStorage 存界面状态)。

**区分情况**: 如果项目名字还在但图片没了 → 图片 URL 过期, 需要从项目文件夹重新读。如果所有东西都没了 → 没做持久化, 让 AI 加。

## 4. 部署后功能不工作

**你看到的**: 本地能用, 部署到网上后某些按钮点了没反应。

**怎么办**: 告诉 AI: "部署后功能失效, 检查是否需要 HTTPS"。部分浏览器功能 (如 File System Access API) 只在 HTTPS 或 localhost 下工作。

**关键信号**: 本地能用, 线上不能用 → 浏览器安全限制。

## 5. 改了这里, 那里崩了

**你看到的**: 你让 AI 修复 A 功能, 修好后 B 功能坏了。

**怎么办**: 告诉 AI: "B 功能被你改坏了, 检查最近的改动"。然后让 AI 运行 `npm run typecheck` 和 `npm run lint`, 它会自动发现遗漏的引用和类型错误。

**关键信号**: 修一个坏一个 → 依赖没更新 → 让 AI 检查所有引用。

## 万能处理流程

```
1. 你发现不对 → 截图或描述现象
2. 告诉 AI → AI 分析原因
3. AI 修改 → 你测试
4. 好了 → 结束
5. 不好 → 回到第 2 步, 换一种描述
```

如果 AI 三次都修不好, 告诉它: "这个已经修了 X 次了, 请换一个思路, 不要继续原来的方案"。

## 当你不知道怎么说的时候

用这个模板:

> "我在做 [什么操作] 的时候, 预期是 [什么效果], 但实际看到的是 [什么现象]"

例如:

> "我在缩放图片的时候, 预期是图片围绕中心放大, 但实际看到的是图片位置跳到角落了"

这样 AI 能立即理解问题。
