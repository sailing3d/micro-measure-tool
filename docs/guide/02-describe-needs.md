# 第二章: 描述你的需求

好的需求描述 = AI 理解准确 = 少走弯路。

## 一句话定义项目

先写一句话, 说清楚这个软件是干什么的:

> "显微镜图像归集与测量 Web 应用"

这句话就够了。不要求你写 "基于浏览器端 File System Access API 的高性能 2D 图像处理工具"——那是 AI 在后续对话中自己搞清楚的。

## 用场景描述功能

不要说 "需要一个格子系统", 要说你要怎么用:

> "多张显微镜照片排列在一个版面上, 格子数量可以调整, 页面可能超过一个屏幕宽度, 但需要方便拖动, 比如右键拖动"

这个描述包含了三个重要信息:
1. 多个图片排列 (不是单图)
2. 格子可调整
3. 超出屏幕, 需要拖动

如果你只说 "要一个格子", AI 可能会问 5 个问题才搞清楚。

## 特殊功能要画清楚

本项目有一个特殊测量工具叫 "限定圆", 如果只写 "先画辅助线, 然后生成圆", AI 会搞错。实际描述是:

> "先画辅助轨迹线。移动鼠标: 沿轨迹线方向移动 = 圆心沿轨迹滑动, 垂直于轨迹方向 = 调整半径。点击左键确认。"

这个描述把交互分成了两个维度 (平行/垂直), 让 AI 能准确实现。

## 画草图比写字快

如果你不擅长组织文字, 直接画草图、拍照片, 发给 AI 说 "做成这样的"。AI 能理解图片中的布局结构。

## 实际案例

这是本项目主人公开头的需求描述 (精简版):

> 前端 Web 应用, 用于处理显微镜拍摄的图像的归集和测量。本地文件夹读写权限。建立新项目, 拖放图片并命名。自定义行列的格子系统, 格子内可移动、旋转、缩放图片, 格子间可移动或交换。像素和微米数比值标定, 用于测量。测量工具包含限定圆工具。测量结果按照图片 → 名称 → 结果显示。保存到项目文件。测量线可删除, 暂时不考虑修改。

这段 150 字的描述, 包含了 10 个功能点。AI 据此进行了 6 轮提问澄清, 产出了 280 行的需求文档。
