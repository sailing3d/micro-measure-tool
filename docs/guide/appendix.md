## B: 技术术语小词典

| 术语 | 解释 |
|------|------|
| 前端 | 在浏览器里运行的软件, 不需要安装 |
| 纯前端 | 没有服务器, 所有功能都在浏览器完成 |
| 框架 | 别人写好的代码库, 帮你快速搭骨架 |
| React | 一个前端框架, 帮你组织界面 |
| zustand | 管理应用数据的轻量工具 |
| Konva | 在网页上画图的库, 支持 Canvas |
| Canvas | HTML5 的画布元素, 可以编程画图 |
| File System Access API | 浏览器读写本地文件夹的功能 |
| 构建 | 把源码转成浏览器能高效运行的版本 |
| 部署 | 把构建好的应用放到网上 |

## C: 本项目的关键决策记录

| 决策 | 原因 |
|------|------|
| 不用服务器, 全靠前端 | File System Access API 直接读写本地文件 |
| 用 zustand 而不是 Redux | 更简单, AI 推荐 |
| 用 Konva 而不是纯 DOM | 图像操作和测量工具需要 Canvas 性能 |
| 用 Tailwind CSS | AI 推荐, 写样式快 |
| 用 Cloudflare Pages | 免费, 自动 HTTPS |

## D: 对话量统计

| 阶段 | 大约对话轮次 |
|------|------------|
| 需求澄清 | 10-15 轮 |
| 架构设计 | 5-8 轮 |

| **总计** | **约 60-80 轮** |

平均每轮你的输入是 1-2 句话或一个选择。
