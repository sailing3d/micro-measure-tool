# 产品需求文档

## 1. 概述

Micro Measure Tool 是一个浏览器端 Web 应用, 用于归集显微镜拍摄的图像并在排版画布上进行测量. 应用通过 [File System Access API](https://developer.mozilla.org/en-US/docs/Web/API/File_System_Access_API) 读写本地文件夹.

## 2. 核心概念

### 2.1 项目 (Project)

- 项目对应一个**本地文件夹**
- 文件夹包含: 所有使用的图片文件 + 一个项目数据 JSON 文件
- 存在一个**全局项目索引** (localStorage), 记录所有项目的路径和元信息
- 支持 IndexedDB 持久化项目句柄, 刷新后自动恢复
- 支持: 新建项目, 打开已有项目, 运行时切换项目

### 2.2 画布 (Canvas)

- 画布是放置图片的背景区域, 使用 `Konva.js` 渲染
- 画布划分为**格子 (cells)**, 按行 (rows) 列 (columns) 排列
- 行列数, 格子大小均可调整
- 画布可能超出屏幕, 使用**右键拖动**平移视角

### 2.3 图片 (Image)

- 图片从文件系统拖入, 直接拖到目标格子中
- 所有图片被复制到项目文件夹内
- 默认使用原始文件名, 支持重命名
- 每个图片占据一个格子

### 2.4 标定 (Calibration)

- 像素到微米的比例比值, **全局共享**所有图片
- 设置方式: 在任意图片上画已知实际长度的线段 + 输入微米值, 或直接输入比例数值
- 标定值与显示缩放比例无关

### 2.5 显示缩放 (Display Zoom)

- 控制所有图片在格子中的视觉大小, **全局一致**
- 初始: 以第一张图 `cover` 方式填充格子的比例为基准
- 动态响应: 窗口大小变化, 格子参数修改时自动重算
- 工具栏 Zoom 控件可手动调节, `↺` 恢复默认
- 缩放不影响标定值 (px 始终指真实图片像素)

## 3. 功能需求

### 3.1 项目管理

| 功能 | 描述 |
|------|------|
| F1.1 新建项目 | 通过 File System Access API 选择或创建本地文件夹作为项目根目录 |
| F1.2 打开项目 | 选择已有项目文件夹 |
| F1.3 启动选择 | 应用启动时弹出: 新建项目 / 打开已有项目 |
| F1.4 运行时切换 | 在 UI 中关闭当前项目, 打开另一个 |
| F1.5 自动保存 | 测量数据, 布局修改后实时写入项目 JSON |

### 3.2 图片管理

| 功能 | 描述 |
|------|------|
| F2.1 拖入图片 | 从系统文件管理器拖拽图片到目标格子 |
| F2.2 文件格式 | 支持常见图像格式 (jpg, png, tiff, bmp 等) |
| F2.3 图片复制 | 拖入时自动复制到项目文件夹, 保留原文件 |
| F2.4 图片命名 | 默认使用原始文件名, 支持在 UI 中重命名 |
| F2.5 图片显示 | 在格子内以全局缩放比例显示 |

### 3.3 格子系统

| 功能 | 描述 |
|------|------|
| F3.1 行列设置 | 工具栏中设置行数, 列数 |
| F3.2 格子大小 | 工具栏中设置格子宽高 (像素) |
| F3.3 响应式重算 | 格子参数或窗口变化时, 显示缩放自动重新计算 |
| F3.4 右键平移 | 在画布上右键拖动平移整个画布 |
| F3.5 图片操作 | 格子内: 移动, 旋转, 缩放单个图片 |
| F3.6 格子间操作 | 两张图片交换位置, 或移动图片到空格子 |
| F3.7 大小不一致 | 鲁棒处理不同尺寸的图片: 较图片以格子范围 `contain` 显示, 小图片居中 |

### 3.4 测量工具

#### 3.4.1 H 型长度测量

| 功能 | 描述 |
|------|------|
| F4.1 画测量线 | 在图片上绘制 H 型 (工字型) 测量线段, 记录像素长度 |
| F4.2 数值换算 | 根据全局标定比例将像素长度换算为微米 |
| F4.3 结果记录 | 保存线段位置, 像素长度, 实际长度 (微米) |

#### 3.4.2 限定圆测量

| 功能 | 描述 |
|------|------|
| F4.4 画辅助轨迹线 | 先在图片上绘制一条线段作为圆心约束轨迹 |
| F4.5 调圆 | 沿轨迹方向移动鼠标 → 圆心沿轨迹滑动; 垂直于轨迹方向 → 调整半径; 实时显示直径 |
| F4.6 确认生成 | 点击左键在当前位置和半径生成圆, 不清除轨迹, 可继续在同一条线上画多个圆 |
| F4.7 记录结果 | 保存圆心位置, 直径 (微米), 实时显示直径标签 |
| F4.8 复用轨迹 | 限定圆工具激活时悬停已有轨迹线 → 高亮 → 点击直接进入调圆阶段 |

#### 3.4.3 工具扩展性

| 功能 | 描述 |
|------|------|
| F4.9 工具注册 | 测量工具模块化, 支持后续添加新工具类型 |
| F4.10 ESC 退出 | 按 ESC 取消当前标定或退出测量工具 |

### 3.5 测量结果

| 功能 | 描述 |
|------|------|
| F5.1 组织方式 | 图片 -> 测量名称 -> 测量结果, 在侧面板以树形显示 |
| F5.2 默认名称 | 测量名称默认为序号 (如 "测量 1", "测量 2") |
| F5.3 结果显示 | 显示测量类型, 数值, 单位 (微米) |
| F5.4 删除测量 | 可删除已创建的测量线和结果 |
| F5.5 不修改 | 暂不支持修改已有测量数据 |

### 3.6 数据持久化

| 功能 | 描述 |
|------|------|
| F6.1 项目 JSON | 每个项目文件夹下含 `project.json`, 存储所有布局, 图片, 测量数据 |
| F6.2 全局索引 | 独立 JSON 记录所有项目路径和元信息 |
| F6.3 实时保存 | 关键操作 (添加图片, 完成测量, 修改布局) 后自动写入 |

## 4. 界面布局

```
+--------------------------------------+
|  工具栏 (Top Toolbar)                  |
|  项目切换 | 网格参数 | 标定 | 测量工具 |
+--------------------------------------+
|                              | 侧面板 |
|  主画布 (Canvas)              | (Side)|
|  +---+---+---+               | 图片   |
|  | A | B | C |               | 列表   |
|  +---+---+---+               |--------|
|  | D | E |   |               | 测量   |
|  +---+---+---+               | 结果   |
|                              | 树形   |
+--------------------------------------+
```

- 工具栏: 项目按钮, 行列设置, 格子大小, 测量工具切换, 标定按钮
- 主画布: Konva Stage, 按格子渲染图片, 支持右键拖动平移
- 侧面板: 上下分区 — 图片列表 (可重命名) + 测量结果树

## 5. 交互

### 5.1 画布导航

| 操作 | 行为 |
|------|------|
| 右键拖动 | 平移画布 |
| 滚轮 | 缩放画布 (朝光标方向, 0.1x – 5x) |

### 5.2 图片操作

| 操作 | 行为 |
|------|------|
| 拖入文件 | 图片放入目标格子 |
| 左键拖动 (图片内) | 移动图片在格子内的位置 |
| 滚轮 (图片上) | 缩放图片 |
| 右键拖动 (图片上) | 旋转图片 |
| 拖拽到另一格子 | 交换位置或移动到空格子 |

### 5.3 测量流程

**H 型长度测量:**
1. 选择测量工具 (H)
2. 在目标图片上按下左键开始画线
3. 拖动调整, 松开完成
4. 测量线留在图片上, 结果计入侧面板

**限定圆测量:**
1. 选择限定圆工具
2. 在图片上画辅助轨迹线 (按下→拖动→松开)
3. 鼠标进入调整模式: 沿轨迹移动 = 移动圆心, 垂直移动 = 调整半径
4. 点击左键确认, 生成圆
5. 圆的图形留在图片上, 结果计入侧面板

## 6. 数据格式

### 6.1 项目 JSON (`project.json`)

```json
{
  "version": 1,
  "name": "项目名称",
  "grid": { "rows": 3, "cols": 4, "cellWidth": 800, "cellHeight": 600 },
  "calibration": { "ratio": 0.5 },
  "displayZoom": 1.0,
  "images": [
    {
      "id": "img-1",
      "filename": "sample_001.jpg",
      "cellIndex": 0,
      "offsetX": 0,
      "offsetY": 0,
      "rotation": 0,
      "scale": 1.0,
      "measurements": [
        {
          "id": "meas-1",
          "name": "测量 1",
          "type": "h-line",
          "points": [ [100, 200], [300, 200] ],
          "lengthPx": 200,
          "lengthUm": 100.0
        },
        {
          "id": "meas-2",
          "name": "测量 2",
          "type": "constrained-circle",
          "trajectory": [ [50, 100], [250, 100] ],
          "center": [150, 120],
          "radiusPx": 40,
          "diameterUm": 40.0
        }
      ]
    }
  ]
}
```

### 6.2 全局索引

```json
{
  "projects": [
    { "name": "项目A", "path": "/Users/xxx/project-a" },
    { "name": "项目B", "path": "/Users/xxx/project-b" }
  ]
}
```

### 6.3 导出格式

**Markdown:**

```markdown
# 项目名称 — 测量数据

## sample_001.jpg
| 名称 | 类型 | 值 |
|------|------|-----|
| 测量 1 | 长度 | 100.00 µm |
| 测量 2 | 限定圆直径 | 40.00 µm |

## sample_002.jpg
| 名称 | 类型 | 值 |
|------|------|-----|
| 测量 1 | 长度 | 85.50 µm |
```

**CSV:**

```csv
图片,测量名称,类型,值,单位
sample_001.jpg,测量 1,长度,100.00,µm
sample_001.jpg,测量 2,限定圆直径,40.00,µm
sample_002.jpg,测量 1,长度,85.50,µm
```

### 3.7 导出

| 功能 | 描述 |
|------|------|
| F7.1 Markdown 导出 | 将当前项目所有测量数据导出为 Markdown 文件 (UTF-8 BOM), 按图片分组表格 |
| F7.2 CSV 导出 | 导出为 CSV 文件 (UTF-8 BOM), 每行一条记录 |
| F7.3 导出入口 | 工具栏导出下拉菜单, 选择格式后通过 File System Access API 保存 |
| F7.4 PNG 导出 | 将完整画布 (含图片+标注, 不含 UI) 以 2x 分辨率导出为 PNG |

## 7. 非功能需求

| 类别 | 要求 |
|------|------|
| 浏览器 | Chrome / Edge (File System Access API) |
| 性能 | 10 张图片以下流畅交互 |
| 离线 | 纯前端, 无网络依赖 |
| 扩展性 | 测量工具模块化, 新增工具无需改核心逻辑 |
